Build a full-stack web application that aggregates tech events (hackathons, webinars, workshops) by scraping multiple platforms, primarily Unstop and similar public event websites.

## Core Goal

Create a backend-driven system that:

* Scrapes event listings from multiple sources (starting with Unstop)
* Normalizes and stores the data
* Exposes APIs to fetch and filter events
* Displays events on a simple frontend (UI decisions handled automatically)

---

## Tech Stack

* Backend: Node.js (Express)
* Scraping: Playwright (preferred) or Puppeteer
* Database: PostgreSQL
* ORM (optional): Prisma or Sequelize
* Deployment-ready structure

---

## Core Features

### 1. Scraper Engine (Modular Design)

Create a modular scraping system where each source has its own scraper module.

Example structure:

* /scrapers/unstopScraper.js
* /scrapers/devpostScraper.js (placeholder)
* /scrapers/eventbriteScraper.js (placeholder)

Each scraper must:

* Visit the events listing page (e.g., Unstop hackathons page)
* Handle dynamic loading (scrolling / pagination)
* Extract structured event data

---

### 2. Unstop Scraper Logic

Use Playwright to:

1. Open:
   https://unstop.com/hackathons

2. Wait for page to load fully

3. Implement auto-scroll to load all events:

   * Scroll until no new content loads

4. Extract for each event:

   * title
   * event_url
   * image (if available)
   * deadline / last date
   * mode (online/offline)
   * tags (if available)
   * organizer (if visible)

5. Return data as JSON array

---

### 3. Data Normalization Layer

All scrapers must output data in this unified format:

```json
{
  "title": "",
  "platform": "unstop",
  "type": "hackathon | webinar | workshop",
  "url": "",
  "image": "",
  "start_date": "",
  "end_date": "",
  "mode": "online | offline | hybrid",
  "tags": [],
  "organizer": "",
  "location": ""
}
```

Implement a function:
normalizeEvent(rawEvent, source)

---

### 4. Database Design (PostgreSQL)

Create a table:

events:

* id (primary key)
* title
* platform
* type
* url (unique)
* image
* start_date
* end_date
* mode
* tags (JSON or array)
* organizer
* location
* created_at
* updated_at

Ensure:

* Duplicate prevention using unique URL
* Upsert logic (insert or update)

---

### 5. Scraping Scheduler

Implement a scheduler:

* Runs every 6–12 hours
* Triggers all scrapers
* Stores fresh data in DB

Use:

* node-cron

---

### 6. API Layer

Create REST APIs:

GET /events

* Fetch all events
* Support query params:

  * type
  * platform
  * tag
  * mode

GET /events/:id

* Fetch single event

POST /scrape

* Manually trigger scraping

---

### 7. Performance Considerations

* Use concurrency carefully in scraping
* Add retry logic for failed pages
* Use rate limiting to avoid blocking
* Cache responses (optional)

---

### 8. Error Handling

* Handle:

  * page load failures
  * selector changes
  * empty results
* Log all scraping errors

---

### 9. Extensibility

Design system so new scrapers can be added easily:

* Common interface:
  scrape(): Promise<Event[]>

---

### 10. Important Constraints

* Only scrape publicly available data
* Do not bypass authentication walls
* Keep scraping frequency reasonable

---

## Final Output

* Working backend server
* Scraper modules
* PostgreSQL integration
* API endpoints
* Basic frontend showing event list (auto-generated by Replit)

The system should be scalable and structured like a real production service.
